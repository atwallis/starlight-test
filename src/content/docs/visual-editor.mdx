The visual editor is a companion application to the Restura Engine that allows you to visually create and modify the database tables and endpoints found in the `restura.schema.json` file.

## Clone the Restura UI repository

Currently, there is no standalone application binary for the Restura UI. You will need to download and run the application locally.

Start by cloning the repository:

export const resturaCloneSSHCode = `
git clone https://github.com/redsky-engineering/restura.git
`;

export const resturaCloneHTTPSCode = `
git clone git@github.com:redsky-engineering/restura.git
`;

With the application running, you can now access the visual editor at <a href="http://localhost:3000" target="_blank" rel="noopener noreferrer">http://localhost:3000</a> in the browser of your choice.


## Sign In
When you first open the Restura UI, you will be prompted to enter an auth token. Enter in the auth token and click the "SIGN IN" button.

<Aside type="note">
The auth token you enter must match the auth token set in the `restura.config.mjs` file.
</Aside>

<Aside type="caution">
The Restura UI connects to the Restura Engine on port 3001 by default. If you cannot login, make sure the Restura Engine is running and accessible at this port.
</Aside>

## Database Tables

When you login, you will be navigated to the "Database" tab. Here you can view the database tables that are managed by Restura.

## API Endpoints

To view endpoints, click on the "API" tab. Here you can view the endpoints that are managed by Restura.

## Apply Changes

When you make changes to the database tables or endpoints, changes must be confirmed to take effect.
- Click "Preview Schema" in the upper right corner to see the changes that will be applied
- Click "Submit" to apply the changes

Upon submission, the changes will be applied to the `restura.schema.json` file and the Restura Engine will automatically restart to apply the changes.
