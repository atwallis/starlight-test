---
title: Parameter Validation
description: Parameter validation in endpoints
---

-   **Validator** – A rule that validates request parameters before processing.
-   **Type check** – A validator that ensures a parameter matches an expected data type.
-   **Min validator** – A validator that enforces a minimum value for numbers.
-   **Max validator** – A validator that enforces a maximum value for numbers.
-   **One of validator** – A validator that restricts values to a specific allowed set.
-   **Request validation** – The process of checking incoming data against defined validators.
-   **Response validation** – The process of verifying outgoing data matches expected types.
-   **Coercion** – The automatic conversion of query parameters to their expected types.
