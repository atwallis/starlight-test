---
title: Authenication & Authorization
---

-   **Role** – A permission level assigned to users that controls access to routes and data.
-   **Scope** – A granular permission that can be combined with roles for fine-grained access control.
-   **Public route** – A route with no roles or scopes, accessible without authentication.
-   **Protected route** – A route requiring specific roles or scopes for access.
-   **Authentication** – The process of verifying a user’s identity and extracting their credentials.
-   **Authorization** – The process of checking if an authenticated user has permission to access a route.
-   **Requester** – The user or system making an API request.
-   **Requester details** – Information about who is making a request, including role, scopes, and user ID.
-   **Auth token** – The secret token required to access Restura’s admin endpoints.
