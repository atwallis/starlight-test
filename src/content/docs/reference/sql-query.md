---
title: SQL Query Building
---

-   **Join** – A SQL operation that combines data from multiple tables based on related columns.
-   **Inner join** – A join that only returns rows where matching records exist in both tables.
-   **Left join** – A join that returns all rows from the left table, with NULL for non-matching right table rows.
-   **Right join** – A join that returns all rows from the right table, with NULL for non-matching left table rows.
-   **Join alias** – A short name assigned to a joined table for referencing its columns.
-   **Where clause** – A condition that filters which rows are returned or affected by a query.
-   **Conjunction** – The logical connector (AND/OR) between multiple where conditions.
-   **Operator** – The comparison type in a where clause: equals, greater than, less than, LIKE, IN, etc.
-   **Order by** – A sorting specification for query results by column and direction.
-   **Sort order** – The direction of sorting: ascending (ASC) or descending (DESC).
-   **Group by** – An aggregation specification that groups rows by a column value.
-   **Subquery** – A nested query within a response that fetches related data from another table.
-   **Assignment** – A value mapping used in INSERT/UPDATE operations to set column values.
