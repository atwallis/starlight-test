---
title: Database
---

-   **Database** – The collection of table definitions in your schema that map to PostgreSQL tables.
-   **Table** – A database table definition containing columns, indexes, foreign keys, and access controls.
-   **Column** – A field within a database table that stores a specific type of data.
-   **Column type** – The data type of a column such as INTEGER, VARCHAR, TEXT, TIMESTAMP, BOOLEAN, or JSON.
-   **Nullable** – A column property indicating whether the field can store NULL values.
-   **Primary key** – A unique identifier column for each row in a table, typically auto-incrementing.
-   **Auto-increment** – A column setting that automatically assigns sequential numeric values to new rows.
-   **Index** – A database optimization that speeds up queries on specific columns.
-   **Unique index** – An index that enforces uniqueness, preventing duplicate values in the indexed columns.
-   **Foreign key** – A relationship between tables where one column references another table’s primary key.
-   **Cascade** – A foreign key action that automatically propagates changes (updates/deletes) to related records.
-   **Check constraint** – A rule that validates column values against a custom SQL expression.
-   **Connection pool** – A managed set of database connections for efficient query execution.
-   **Scratch database** – A temporary database used to calculate migration differences.
